# 语调与文案

简体中文为唯一交互语言。语气冷静、直接、克制——状态描述事实，不寒暄、不致歉、不夸张。

## 大小写

- 主品牌一律写作 **Unif**——不写 `UNIF`、`unif` 或加连字符。
- 子产品形如 `Unif <中文名>`（半角空格分隔）。
- 拉丁字符与 CJK 字符之间使用单个半角空格。

## 人称

- 助手显示为 `AI` 或 `助手`（头像内）。
- 用户显示为 `我`（自己头像内）。
- 正文避免第一/第二人称，直接陈述事实：`已思考 3s`、`正在执行…`、`AI 回复中…`。

## 状态文案速查

| 场景 | 文案 |
|---|---|
| 工具运行中 | `执行中` |
| 工具完成 | `完成` 或 `完成 · 0.6s` |
| 工具失败 | `失败` |
| 推理进行中 | `思考中…` |
| 推理完成 | `已思考 3s` |
| 确认发起 | `操作确认` |
| 确认完成 | `已确认，正在执行…` |
| 确认取消 | `已取消` |
| 发送失败 | `发送失败，请重试` |
| 输入空闲 | `输入消息…` |
| AI 流式回复 | `AI 回复中…` |

## 建议 chip 语气

像用户的口气——疑问句或祈使句，**不加句号**：

- `今天要拜访哪些客户？`
- `帮我查一下今天的计划`
- `新建一个拜访记录`

## 标点

- 全角中文标点：`，` `。` `？` `：`
- 流式/加载用西文 `…`
- 中点 `·` 用于分隔元数据：`客户名称 · 地址`

## Emoji

**禁用所有装饰 emoji。** 包括国旗、贴纸、装饰字符。功能性状态全部走图标系统。如果一个字符不传递信息，它就不应该出现。

```tsx
// ✗ 错误
<Text>🎉 提交成功！</Text>
<Text>📍 当前位置</Text>

// ✓ 正确
<Text>提交成功</Text>
<Text>
  <Icon name="location" size={14} /> 当前位置
</Text>
```
